What We're Building
🎯

The Problem

Most students fail interviews not because of lack of knowledge — but lack of practice. Traditional mock interviews are expensive, inconsistent, and don't scale. IntelliHire solves this with AI.

🤖

The Solution

A 5-module pipeline that mirrors real hiring: ATS screening → Aptitude test → Technical round → HR simulation → Behavioral analysis. Fully voice-enabled, adaptive, and Azure-powered.

🔬

The Tech

Built on Azure OpenAI, Azure Speech, Computer Vision models (MediaPipe, DeepFace, YOLOv8), IRT psychometrics, and NLP pipelines — deployed as a static + FastAPI hybrid on Azure.

📐

The Approach

Every module is research-backed: IRT 3PL adaptive testing, STAR-method NLU, EAR blink/gaze detection, VADER sentiment, and a Confidence Index formula for behavioral scoring.

Regex NER + Section Parser
NLP — Entity Extraction
Rule-based Named Entity Recognition using Python regex patterns. Extracts name, email, phone, LinkedIn, GitHub, education, experience blocks, projects. Section detection via header pattern matching (50+ patterns).
Libraries

TF-IDF Cosine Similarity
NLP — Keyword Matching
Term Frequency–Inverse Document Frequency vectorizes both resume and JD. Cosine similarity gives match percentage. Used for keyword match (30% weight) and experience relevance (20% weight) in ATS score.
Libraries
scikit-learn TfidfVectorizercosine_similarity
📊
Weighted ATS Scorer
Rule-based ML — Composite Scoring
Six-component weighted scoring: Keyword Match (30%), Skills Match (20%), Experience Relevance (20%), Formatting (10%), Readability (10%), Section Completeness (10%). Final 0–100 score = Σ(weight × component).
Formula
ATS = Σ(wᵢ × sᵢ) × 100
📖
Flesch-Kincaid Readability
Statistical NLP
Flesch Reading Ease score measures resume language clarity. Optimal range 60–80 for professional resumes. Penalizes overly complex or overly simple text. Part of 10% readability ATS component.
Library
textstat.flesch_reading_ease()
⚡
Skill Gap Prioritizer
Rule-based Ranking
Ranks missing skills by frequency in JD using regex count. Most-mentioned missing skills ranked highest — e.g., if "Kubernetes" appears 4 times in JD and missing from resume, it's Priority 1 to add.
Libraries
Python re.findall()sorted() by freq
🔤
Fuzzy String Matching
NLP — Approximate Match
rapidfuzz library catches skill aliases: "JS" ↔ "JavaScript", "ML" ↔ "Machine Learning", "K8s" ↔ "Kubernetes". Threshold ≥ 82% similarity treated as a match. Prevents false negatives in skill matching.
Library
rapidfuzz.fuzz.ratio()threshold 82%
ATS Scoring Formula
6-Component Weighted ATS Score
ATS_Score = Σ(wᵢ × sᵢ) × 100
w₁=0.30 × keyword_match ← TF-IDF: resume∩JD keywords / JD keywords
w₂=0.20 × skills_match ← exact + fuzzy skill overlap
w₃=0.20 × exp_relevance ← TF-IDF cosine(experience_text, JD)
w₄=0.10 × formatting ← contact info + length + bullets + dates
w₅=0.10 × readability ← Flesch-Kincaid ease score (optimal 60–80)
w₆=0.10 × section_complete ← found_sections / 6 expected sections

IRT 3-Parameter Logistic
Psychometric Model
Models item probability as a function of student ability θ. Three parameters: a (discrimination), b (difficulty), c (guessing). Allows adaptive selection of questions at the right difficulty level for each student.
Libraries
NumPySciPyCustom IRT engine
🎯
Maximum Fisher Information
Information Theory — Question Selection
Selects the next question by maximizing Fisher Information I(θ) = [P′(θ)]² / [P(θ)(1-P(θ))]. The question with highest information at current θ estimate is chosen — most efficient ability measurement.
Formula
I(θ) = [P′(θ)]² / [P(θ)(1-P(θ))]
🔢
MLE Newton-Raphson
Statistical Optimization
After each answer, updates θ using Maximum Likelihood Estimation with Newton-Raphson iteration. Converges in 2–3 steps. Stops when Standard Error SE(θ) < 0.3, meaning sufficient precision reached.
Library
scipy.optimizeSE(θ) stopping criterion
🧩
Naive Bayes Topic Router
ML Classifier
Before starting adaptive test, classifies student's domain (Math, Verbal, Logical, Data Interpretation) using Naive Bayes on resume skills. Routes question bank to relevant topic. Trained on labelled question corpus.
Library
scikit-learn MultinomialNBTF-IDF features

STAR Method NLU
NLP — Structured Answer Detection
Detects whether answer follows STAR structure: Situation (context keywords), Task (role/responsibility phrases), Action (past-tense verbs), Result (metric/outcome phrases). Regex + classifier hybrid. Score per component.
scikit-learnPython regexNLTK POS tags
😊
VADER Sentiment Analysis
NLP — Tone Scoring
VADER (Valence Aware Dictionary and sEntiment Reasoner) scores each answer for positive/negative/neutral/compound tone. Optimal HR score: high positive compound (0.5–1.0). Anxiety detected via negative sentiment.
NLTK SentimentIntensityAnalyzercompound score
🚫
Filler Word Detector
NLP — Speech Quality
Counts filler words: "uh", "um", "like", "you know", "basically", "right". Regex scan on STT transcript. Score = 100 − (filler_count × 5). More than 10 fillers in a 2-min answer = significant penalty.
regex word boundaryfiller_count × 5 penalty

Azure AI Language's RoBERTa-based NLI model classifies answer as entailment/contradiction/neutral vs ideal answer. Used for deep semantic relevance beyond cosine similarity. Catches paraphrased correct answers.

EAR Gaze + Blink Detection
CV Algorithm
Eye Aspect Ratio formula using 6 dlib landmarks per eye. EAR < 0.2 = blink. Eye direction vector from iris center relative to eye corners determines gaze. Optimal blink rate: 12–20 per minute.
EAR = (|p2-p6|+|p3-p5|) / (2|p1-p4|)
🗺️
MediaPipe FaceMesh
CV — Landmark Detection
Google's BlazeFace + mesh CNN detects 468 (x,y,z) face landmarks in real time. Runs in browser via TensorFlow.js — no server round trip. Used for gaze direction, head pose, blink detection.
MediaPipeTensorFlow.js (browser)468 landmarks
🧍
YOLOv8 Pose Estimation
Deep Learning — Body Language
Detects 17 body keypoints per frame: shoulders, elbows, wrists, hips, knees. Posture score = shoulder symmetry × 40 + lean_score × 40 + arm_openness × 20. Penalizes slouching or closed body language.
YOLOv8n-pose17 keypointsUltralytics Python
😐
DeepFace 7-Emotion Model
Deep Learning — Emotion
VGG-16 CNN trained on FER2013 detects 7 Ekman emotions per frame: Happy, Neutral, Surprise, Sad, Angry, Fear, Disgust. Emotion stability = 100 − (variance × 10). Frequent happy/neutral = best score.
DeepFaceVGG-Face7-class softmax
Confidence Index Formula
Composite Confidence Score (0–100)
CI = 0.30×eye_contact + 0.25×emotion_stability + 0.25×posture + 0.10×smile_freq + 0.10×blink_norm
eye_contact = frames_looking_at_camera / total_frames × 100
emotion_stability = 100 − (variance_in_emotion × 10)
posture = shoulder_sym × 40 + lean_score × 40 + arm_open × 20
blink_norm = 100 − |blink_rate − 16| × 5 ← optimal 12–20 blinks/min
